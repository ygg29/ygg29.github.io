iOS启动优化-二进制重排

发表于 2020-03-04 | 分类于 iOS开发

随着业务开发进入迭代稳定期，App 的各种优化任务随之而来。抖音研发实践中提出了二进制重排的优化经验，号称提升了 15%的启动速度，本文尝试探究一下二进制重排的原理及 clang 插桩的优化方案。

启动过程

启动分为两个部分 ： pre-main 和 main 后阶段

启动时间检测：

  1. pre-main

  • 配置 DYLD环境变量检测启动时间：DYLD_PRINT_STATISTICS

    通过控制台输出查看耗时：

    • dylib loading time

      动态库的加载耗时，Apple建议为 6 个，所以如果超过这个数量，最好进行库的合并

    • rebase/binding time

      符号绑定耗时，由 DYLD 实现，和依赖的动态库数量正相关

    • ObjC setup time

      OC class 的注册时间，class 越多越耗时

    • initializer time

      执行 load和构造函数的耗时

  1. main

    main 后阶段可控，耗时可通过埋点或者 instruments 工具监控，和业务相关性较大，在此不做过多说明

二进制重排原理

安全考虑，iOS 使用了虚拟内存，并在iOS4.3 后引入了 ASLR 安全机制。由于启动过程所调用的函数可能在多个 page 中，为了尽量减少 Page Fault 通过重排将启动时期所调用的函数尽量排在同一个 page 中，进而减少启动耗时。

Clang插桩

二进制重排的难点在于获取启动过程中调用的函数/Block/方法的符号，抖音方案也仅覆盖80%～90%的符号，clang插桩是个更有效更简单的方式。
